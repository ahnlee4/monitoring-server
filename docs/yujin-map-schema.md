# YujinCombine Address Map

## 기준 원본

- `DatabaseHelper.setInsertData()`
- `DataValue`
- `EventMap.REQUEST_ADDRESS`
- `Constants`

웹 포팅에서는 이 주소 체계를 새로 정의하지 않고 그대로 따라야 한다. 이 문서는 그중 실제 구현에 바로 필요한 축만 요약한다.

## 주소군

- `00xx`
  시스템 공통 상태/설정
- `01xx`
  네트워크 및 제품 설정
- `11xx` ~ `1FA4`
  인젝션 계열 장비 15대 템플릿
- `21xx` ~ `2FA4`
  오일프리 계열 장비 15대 템플릿
- `E0xx` ~ `EFxx`
  DIO 영역
- `F0xx` ~ `FFxx`
  모듈/아날로그 확장 영역
- `31` ~ `38`
  전력계 요청 주소로 예약

## 핵심 시스템 주소

- `0000` 서비스 압력
- `0002` 연결된 컴프레서 비트
- `0006` 인젝션/오일프리 선택 비트
- `0016` 언로드 압력
- `0018` 로드 압력
- `001E` DIO 연결 상태
- `0020` 4-20mA 연결 상태
- `0022` 전체/개별 모드
- `0028` ~ `0038` 운전 순서
- `004A` 옵션 장치 비트
- `004C` 사용 장치 비트
- `005A` 시스템 제어
- `005C` ~ `0062` 현재 시각

## 장비 템플릿 의미

### 인젝션 `1nxx`

주요 오프셋:

- `00` 압력
- `02` 온도
- `04` 인버터 RPM
- `0A` 알람
- `0C`, `0E` 고장 플래그
- `16` CP 상태
- `18` 운전 모드
- `20`, `22`, `24` 제어 압력 계열
- `26`, `28` 언로드/로드 압력
- `68`, `6A` 총 운전시간 L/H
- `72`, `74` 모델 정보
- `7C`, `7E`, `80` 시리얼 정보

### 오일프리 `2nxx`

주요 오프셋:

- `00`, `02`, `04`, `06` 압력류
- `0C` ~ `14` 온도류
- `28` 알람
- `2A`, `2C`, `2E` 고장 플래그
- `30` CP 상태
- `38` 인버터 RPM
- `3A` 운전 모드
- `46`, `48`, `4A` 제어 압력류
- `4E`, `50` 언로드/로드 압력
- `9A`, `9C` 총 운전시간 L/H
- `7C`, `7E`, `80` 모델/버전

## 요청 주소 맵

`EventMap.REQUEST_ADDRESS` 기준:

- `0 -> 00`
- `1 -> 01`
- `2..19 -> 11..22`
- `20..35 -> E0..EF`
- `36..51 -> F0..FF`
- `52..59 -> 31..38`

즉 Android 런타임의 `requestEquipComm[]` 인덱스와 실제 RS485/TCP 요청 주소는 1:1로 매핑된다.

## 웹 포팅에 필요한 해석 규칙

- `DataValue.getMapDataValue(equipNum, lowId)` 가 사실상 공용 조회 API 역할을 한다.
- `0006` 비트를 보고 장비가 인젝션인지 오일프리인지 판정한다.
- `0008` 비트를 보고 화면에서 숨길 장비를 판정한다.
- `0028` 이후 운전 순서값으로 메인 화면의 장비 정렬 순서가 결정된다.
- `004A`, `004C` 비트로 옵션/사용 장치 표시가 갈린다.
- `DataValue.requestEquipComm[]` 와 `EventMap.REQUEST_ADDRESS` 조합이 통신 상태 UI에 직접 반영된다.
